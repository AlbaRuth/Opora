# Alembic Migrations for Opora

## Overview

This directory contains database migrations using Alembic. Migrations allow version control for your database schema, enabling safe schema evolution and team collaboration.

## Quick Commands

```bash
# Check current migration status
alembic current

# Show migration history
alembic history

# Upgrade to latest version
alembic upgrade head

# Downgrade one revision
alembic downgrade -1

# Create a new migration (after modifying models)
alembic revision --autogenerate -m "description"

# Stamp existing database with current revision (for existing DBs)
alembic stamp head
```

## Workflow

### 1. Development

When you modify database models in `db/models/`:

```bash
# Generate migration automatically from model changes
alembic revision --autogenerate -m "add user profile fields"

# Review the generated migration file in alembic/versions/
# Edit if necessary (e.g., add data migrations, indexes)

# Apply migration to local database
alembic upgrade head

# Run tests to verify
pytest
```

### 2. Production Deployment

```bash
# Before deployment - create backup
pg_dump $DATABASE_URL > backup_$(date +%Y%m%d_%H%M%S).sql

# Run migrations
alembic upgrade head

# Verify
alembic current
```

## Migration Guidelines

1. **Always review autogenerated migrations** - Alembic detects most changes, but some require manual adjustments
2. **Make migrations reversible** - Ensure `downgrade()` is properly implemented
3. **Test on staging first** - Never run untested migrations on production
4. **Keep migrations small** - Prefer multiple small migrations over one large one
5. **Data migrations** - Use `op.execute()` for data transformations in separate migrations

## Troubleshooting

### Database out of sync

If models and database schema diverge:

```bash
# Check differences (dry run)
alembic revision --autogenerate --sql

# Or use stamp for existing databases
alembic stamp <revision_id>
```

### Migration conflict

When multiple developers create migrations:

```bash
# Ensure single head
alembic heads

# If multiple heads, merge them
alembic merge -m "merge heads" <revision1> <revision2>
```

## Configuration

- `alembic.ini` - Main configuration file
- `alembic/env.py` - Environment setup, connects to Opora's database config
- Uses async PostgreSQL driver (asyncpg) for non-blocking migrations
